# Master Seal Card

**Seal:** ATM-Sphere v1.17r0
**Realm:** ATM-only
**Geometry:** R=15000 m, flattening=0
**Tick:** Δt=0.01 s (100 Hz)
**Gravity:** g₀=9.80665 m/s² (constant)
**Ceiling:** ATM_TOP=8000 m, SOFT=100 m
**Atmosphere:** still air; constant density ρ₀=1.225 kg/m³ (B1/B2 energy model; ISA-vs-altitude deferred to B5)
**Determinism:** det_math only; ban libm/fast-math/FMA/x87
**Wire/Hash:** GEO-001 (lat/lon×1e7, bearing×1e6, h×1e3; ZigZag+LEB128) + **KIN-002** aux block (phi×1e6, tas×1e3, **gamma×1e6**) + **WEAPON-001** aux block (per-aircraft **hp×1e3**, **fire_cd×1e3**, **ammo×1e0**, **last_hit_by×1e0**; per live round: GeoPoint + **damage×1e3** + **ttl**/**owner** exact i64) — snapshot **protocol 6** (v1.12r0 added the gunnery state to the wire so MP can replicate rounds/HP/kills; **v1.14r0 added the magazine `ammo`** at unit scale so a remote client shows a rounds-remaining counter — protocol 4→5; **v1.17r0 added attacker attribution `last_hit_by`** at unit scale, ZigZag carrying the -1 "never hit" sign, so a remote client renders an attributed kill-feed — protocol 5→6; throttle/target_g/fire are inputs, not state). The wire is lossy/downstream — the world_hash source of truth stays `Kernel::snapshot()`.
**Flight model:** B3 limits & stall (B2 lift & pitch) — full **3-DOF point mass**. Flight-path angle **γ is a stored state** (per-aircraft state is now the 7-tuple `lat,lon,psi,phi,alt,tas,gamma`), driven by commanded load factor `n` (`Command.target_g`). Altitude is **earned** (`alṫ = V·sin γ`); pulling g raises induced drag → bleeds speed. Generalizes B1 (`n=1/cosφ, γ=0` ⇒ the old `ψ̇=g₀tanφ/V` level turn). Builds on B1 energy (TAS from thrust−drag). V_MIN=30 m/s. **B3 (v1.7r0):** the achievable `n` is bounded per-airframe by BOTH the structural g limits (`n_min_struct`/`n_max_struct`) AND the C_Lmax aerodynamic ceiling `n_aero = cl_max*qS/(m*g0)` — `n = clamp(n_cmd, max(n_min_struct,-n_aero), min(n_max_struct,n_aero))`. Below the corner speed `V* = stall*sqrt(n_max_struct)` the aero ceiling binds and the turn collapses (**accelerated stall**); the 1 g stall speed falls out for free. Retires the B2 global placeholder [-3,9]; post-stall lift held at C_Lmax (no departure). No new det_math (det_sin/det_cos only). **B4 (v1.8r0):** **per-airframe aero retune (data-only mechanism, but moves goldens → reseal).** The envelopes carried the un-calibrated v1.2r0 aero, so emergent level top speeds ran ~30 % low (109–142 m/s). B4 lifts them to **historical** values (148–195 m/s ≈ 533–702 km/h) by moving **only `thrust_static_n`/`v_max_mps` per airframe** (solved via `tools/b4_retune.py`); every turn/stall-defining param (`mass`,`wing_area`,`cl_max`,`induced_k`,`n_max/min_struct`,`stall`, all LUTs) is **fixed**, so the iconic turn ordering (A6M2/Spitfire best, P-47 worst) and the B3 C_Lmax↔stall coherence are preserved exactly. `v_max_mps` is now the flat-curve thrust-zero asymptote (~700, not a physical Vne); `v_ne_mps` is validation-only (never loaded by the kernel). No new det_math, no state/wire change. Emergent perf table: `tools/perf_probe.py`.
**Weapons:** **Step 7 guns G1→G3 COMPLETE (v1.9r0 / v1.10r0 / v1.11r0).** A fired round is a deterministic point mass on the sphere — the **n=0 / thrust=0 specialization of the aircraft 3-DOF step** (gravity arcs γ down; lumped quadratic drag k=2e-4; heading fixed; integer ttl 250; despawns on ttl/ground). **G2 hit/damage:** a round within **HIT_RADIUS 60 m** (horizontal great-circle via the spherical **law of cosines** vs `cos(HIT_RADIUS/R)` — **acos monotone ⇒ no det_acos**) AND **HIT_ALT_GATE 60 m** of an **alive** enemy (≠ firer) deals damage and despawns; **hp≤0 == dead** ⇒ flight frozen, can't fire, excluded from hits (rounds pass through). **G3 per-airframe roster + fire-rate:** the gun globals moved into the envelope — **hp_start** (toughness), **muzzle_v_mps**, **damage_per_round** (carried by the round), **rof_interval_ticks** (fire-rate, gated by a per-aircraft **fire_cd** cooldown, decrement-then-fire). So .50-cal platforms (P-47D/P-51) = high rate + durable; 20-mm cannons (A6M2/La-7) = high damage/round; A6M2 = glass cannon. **NO new det_math** across all three (det_sin/det_cos + ÷×− only). Canonical snapshot now: aircraft **9×f64** (…,gamma,**hp**,**fire_cd**); projectile **7×f64** (…,gamma,**damage**) + u32 ttl + u32 owner. `Command.fire` is the trigger. Per-airframe constants live in the envelopes (data); drag/ttl + the no-arg START_HP stay global. **Wire transport (v1.12r0):** the gunnery state now rides the **WEAPON-001** snapshot section (protocol 3→4) — per-aircraft hp/fire_cd + the live rounds (position/damage/ttl/owner) — so multiplayer/replay can DRAW the dogfight from the wire (the `seads_record --gundemo` viewer sources HP bars + tracer rounds + kills from the decoded wire). Transport-only: hp/fire_cd/damage already live in the canonical state ⇒ **all 9 goldens byte-identical** across the reseal.
**Ammunition (G4, v1.13r0):** **Step 7 guns G4 COMPLETE — the gunnery model is closed.** Each envelope now carries **ammo_start** (the per-airframe magazine, abstract rounds). Firing is gated on **ammo > 0** as well as the fire-rate cooldown; a shot spawns the round, decrements ammo (clamped at 0), and resets the cooldown. An empty magazine falls silent (**"Winchester"**) — the trigger may stay held and the aircraft alive, but no further rounds spawn and the cooldown is not reset. **ammo is the 10th per-aircraft snapshot f64** (appended after fire_cd) ⇒ all 9 prior goldens moved (ammo constant/identical for the unchanged scenarios) + the NEW **GOLDEN-SK-Winchester-001** (an A6M2 empties its 100-round cannon at tick 891, then silent). Relative WWII endurance: A6M2 fewest (100, the ~60-rpg 20 mm cannon), P-47D most (340, eight .50-cal belts). **STILL no new det_math** (a pure integer-valued counter, like fire_cd); ammo lives in the CANONICAL state and — **since v1.14r0 — also rides the WEAPON-001 snapshot wire** (the 10th per-aircraft field, unit-scale quantized, snapshot protocol 4→5), so a remote/late-join client shows a rounds-remaining counter (exactly as fire_cd became kernel-state at G3 then rode the wire at v1.12r0). Transport-only reseal ⇒ all 10 goldens byte-identical. See ADR-Step7-Guns-G4 (counter) + ADR-Step7-Guns-WireTransport-Ammo-v1.14r0 (wire).
**Convergence (v1.15r0):** **gun harmonization.** Each envelope carries **convergence_m** (per-airframe boresight range). SEADS models a single **centerline** battery, so harmonization is realized as **vertical boresight zeroing** — a fired round's initial **γ is offset UP** by the flat-fire drop-compensation angle **δ = ½·g₀·convergence_m / v²** (v = firer TAS + muzzle_v) so its ballistic trajectory crosses the aim (sight) line at convergence_m instead of always falling below the nose. **Pure ±×÷ ⇒ NO new det_math.** A **kernel spawn-geometry** change (round initial γ) ⇒ moves the **3 firing goldens** (Gunfire/Hit/Winchester); the **7 non-firing goldens are byte-identical**. **No wire change** (γ already rides KIN-002 / the projectile block); session digest moves (SESSION-SK-001 fires), **event digest unchanged** (kill sequence identical). Per-airframe (~180–275 m): p47d 270, p51 275, spitfire/ki61 230, a6m2 250, la7 200, bf109/yak3 180. See ADR-Step7-Guns-Convergence-v1.15r0.
**Attribution (v1.16r0):** **attacker attribution — "who fired the killing round".** Each aircraft carries **last_hit_by** (the index of the aircraft whose round most recently damaged it, or **-1 == never hit**), set at hit time from the striking round's **owner** — the **kernel-side event hook** the earlier guns/netcode layers deferred (layer 6 could DERIVE hit/kill moments from hp deltas but not say WHO). It **persists through death**, so at hp≤0 it names the **killer** (an attributed kill-feed). **last_hit_by is the 11th per-aircraft snapshot f64** (appended after ammo) ⇒ **all 10 goldens move** (last_hit_by = -1 identical for every aircraft never hit; GOLDEN-SK-Hit-001's a6m2 now records last_hit_by = 0, the p47d) — **proven additive**: stripping the 11th f64 from each v1.16r0 golden reproduces its v1.15r0 hash byte-for-byte (attribution perturbs no trajectory). **STILL no new det_math** (a pure integer-valued state, like fire_cd/ammo). **Since v1.17r0 it also rides the WEAPON-001 wire** (the 11th per-aircraft field, unit-scale, snapshot protocol 5→6 — deferred one seal exactly as ammo was kernel-state at G4 then rode the wire at v1.14r0) **and stamps the layer-6 EVENT channel** (`Event.attacker`, the 7th event field — the server reads the target's post-step last_hit_by when it derives a hit/kill from the observed hp delta), so a remote client renders the attributed kill-feed from EITHER path (freshest state frame or exact-sequence event journal). Transport-only reseal ⇒ all 10 goldens byte-identical; session + event digests move. See ADR-Step7-Guns-Attribution-v1.16r0 (hook) + ADR-Step7-Guns-WireTransport-Attribution-v1.17r0 (wire).
